# [Get started with CMake Tools on Linux](https://code.visualstudio.com/docs/cpp/cmake-linux)

In this tutorial, you'll use the CMake Tools extension for Visual Studio Code to configure, build, and debug a simple C++ CMake project on Linux.

- C++ compiler
- C++ debugger
- C++ extension for VS Code

and

- CMake
- CMake Tools extension for VS Code

```bash
$ cmake --version
cmake version 3.23.4
```

![CMake Tools extension for VS Code](cmake.png)

## Create a CMake project

The operations to create a project will create two files and one directory in the directory VS Code opens. So let VS Code open the project directory (`~/proj` for this tutorial):

- `CMAkeLists.txt`
- `main.cpp`
- `build` directory (by `cmake`)

Open the Command Palette and run the `CMake: Quick Start` command:

- select a kit (installed GCC is on the list)
- enter a project name (`hello` for this tutorial)
- select `Executable` as the project type

Then the extension run CMake.

A kit represents a toolchain, which is the compiler, linker, and other tools used to build your project. The kit configuration file `cmake-tools-kits.json` in `~/.local/share/CMakeTools/` directory will be loaded (created the first time you build with CMake Tools).

```
[
    {
        "name": "GCC 10.2.1 aarch64-linux-gnu",
        "compilers": {
            "C": "/usr/bin/aarch64-linux-gnu-gcc",
            "CXX": "/usr/bin/aarch64-linux-gnu-g++"
        }
    }
]
```

The `Quick Start` command also run `cmake`:

```
cmake -S~/proj -B~/proj/build -G "Unix Makefiles"
```

As the `cmake` configuration step generates a lot of files in the build-tree, you should put an `.gitignore` file for CMake such as:


## Files the extension creates

```cmake
cmake_minimum_required(VERSION 3.0.0)
project(hello VERSION 0.1.0)

include(CTest)
enable_testing()

add_executable(hello main.cpp)

set(CPACK_PROJECT_NAME ${PROJECT_NAME})
set(CPACK_PROJECT_VERSION ${PROJECT_VERSION})
include(CPack)
```

The minimum version of CMake should be `3.20.0`:

```cmake
cmake_minimum_required(VERSION 3.20.0)
```

The initial C++ code generated by the `CMake: Quick Start` command is:

```c++
#include <iostream>

int main(int, char**) {
    std::cout << "Hello, world!\n";
}
```

## Kits and variants

The kit you selected previously is shown in the Status bar. For example, `GCC 10.2.1`:

![Status bar](statusbar.png)

A *variant* contains instructions for how to build your project. By default, the CMake Tools extension provides four variants, each corresponding to a default build type:

- `Debug`: Disable optimizations - include debug info
- `Release`: Optimize for speed - exclude debug info
- `MinRelSize`: Optimize for smallest size - exclude debug info
- `RelWithDebInfo`: Optimize for speed - include debug info

To select a variant, open the Command Palette and run the `CMake: Select Variant` command.

## Configure and build the project

Once you've selected a kit and a variant, the `CMake: Configure` command available in the Command Palette will configure your project. This generates build files in the project's build folder using the kit and variant you selected.

After configuring your project, you're ready to build. Open the Command Palette and run the `CMake: Build command`, or select the `Build` button from the Status bar.
